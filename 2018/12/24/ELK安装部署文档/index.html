ELK安装部署文档

Dec 24, 2018 | 运维
文章目录

##ELK权威中文文档 (https://www.elastic.co/guide/cn/index.html)

本教程ELK架构：服务端安装elasticsearch logstash kibana 客户端安装 filebeat 利用filebeat采集日志 发送到服务端logstash，由服务端logstash 统一收集整理发送给服务端es，并由kibana展示。

环境说明：本教程服务端系统为centos7.3 客户端 centos6.8

准备工作：服务端安装 jdk1.8，因为elasticsearch是java开发的。

安装包准备： es：

  • https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-6.5.4.rpm
  • kibana：https://artifacts.elastic.co/downloads/kibana/kibana-6.5.4-x86_64.rpm
  • logstash：https://artifacts.elastic.co/downloads/logstash/logstash-6.5.4.rpm
  • filebeat：https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-6.5.4-x86_64.rpm

###一. 服务端 安装es

  1. 采用rpm包安装 安装包下载地址：https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-6.5.4.rpm
    wget   https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-6.5.4.rpm
rpm -ivh elasticsearch-6.5.4.rpm

2.查看es安装位置

3.配置es
vim /etc/elasticsearch/elasticsearch.yml （修改以下配置）

注意格式 （留一个空格)

  1. 启动 es 加入开机自启

    systemctl start elasticsearch (开启)
systemctl status elasticsearch (状态)
systemctl restart elasticsearch (重启)
systemctl enable elasticsearch (加入开机自启)

  2. 检查es 是否启动 

    netstat -anlptu | grep 9200  #查看端口是否被监听
curl http://localhost:9200     #查看es状态

###二.服务端 安装kibana
1.1 下载安装kibana

wget https://artifacts.elastic.co/downloads/kibana/kibana-6.5.4-x86_64.rpm
rpm -ivh kibana-6.5.4-x86_64.rpm

1.2 查看 kibana 安装路径

1.3 修改配置文件
vim /etc/kibana/kibana.yml # 修改一下项

1.4 启动kibana并加入开机自启
systemctl start kibana #开启
systemctl enable kibana #加入开机自启
systemctl restart kibana #重启
systemctl stop kibana #停止
systemctl status kibana #状态

1.5 检查kibana启动状态
netstat -anlptu | grep 5601 #查看端口是否被监听
浏览器访问 http://192.168.5.242:5601

###三.服务端 logstash 安装
1.1 下载安装logstash
wget https://artifacts.elastic.co/downloads/logstash/logstash-6.5.4.rpm
rpm -ivh logstash-6.5.4.prm

2.2 查看logstash

2.3 配置logstash
vim /etc/logstash/logstash.yml # 修改以下配置

path.data: /var/lib/logstash
http.host: “192.168.5.242”
path.logs: /var/log/logstash

至此 ELK 服务端安装完成 #######

###四.客户端 filebeat 安装
1.1 下载安装filebeat
wget https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-6.5.4-x86_64.rpm
rpm -ivh filebeat-6.4.2-x86_64.rpm

1.2 查看 filebeat 安装路径。

1.3 修改 filebeat 配置文件
vim /etc/filebeat/filebeat.yml
1.4 启动filebeat （centos 6 ）
/etc/init.d/filebeat start
加入开机自启，我采用的方式 在/etc/rc.local 添加 /etc/init.d/filebeat start
这个时候 客户端 就开始给服务端 logstash 发送日志了 ，接下来 我们配置 服务端logstash 用来接收filebeat 发过来的日志 并且 发送到es，通过kibana 展示出来。

###五. 服务端 logstash 配置接受 filebeat 发送的数据。
1.1 编写 日志接收文件。 /etc/logstash/conf.d
vim /etc/logstash/conf.d/monitor-pile.conf # 名字随便写，以.conf 结尾

我的配置如下 ：###########

input {
      beats {
           port=>5044
          # codec => json
      }
}

output {

#############192.168.5.244##########################

if [fields][logtype] == “5.244DataTransServer”{ ##这里的 logtype 是客户端定义好的名字。
elasticsearch {
hosts => [“192.168.5.242:9200”] ##输出到es
index => “datatransserver-%{+YYYY.MM.dd}” ##创建索引，索引名字为 xxx-年月日形式
}
}

if [fields][logtype] == “5.244-module-app”{
elasticsearch {
hosts => [“192.168.5.242:9200”]
index => “module-app-%{+YYYY.MM.dd}”
}
}

}

1.2 测试 logstash 配置是否正确。
/usr/share/logstash/bin/logstash –path.settings /etc/logstash/ -f /etc/logstash/conf.d/log.conf –config.test_and_exit ##测试并退出

测试通过 后， 正式启动 logstash
/usr/share/logstash/bin/logstash –path.settings /etc/logstash/ -f /etc/logstash/conf.d/log.conf & ##放入后台运行
1.3 浏览器 打开kibana 创建索引。

1.4 创建索引后即可查看日志。

###六.定时清理索引。
1.1 查看索引
curl ‘localhost:9200/_cat/indices?v’
1.2 删除索引
curl -XDELETE http://localhost:9200/module-manage-2018.11.11 # 删除module-manage-2018.11.11 这个索引
1.3 查看 es占用空间大小
df -h
du -sh /var/lib/elasticsearch/
1.4 编写脚本 实现定时清理七天以前的索引

############################################################################

/bin/bash

指定日期(7天前)
DATA=date -d "1 week ago" +%Y.%m.%d

当前日期
time=date

删除7天前的日志

############################################################################

###七. nginx 反向代理 kibana，利用htpasswd工具 实现 用户名 密码登陆。
1.1 安装nginx （安装过程省略）
1.2 nginx 配置

worker_processes  1;
events {
    worker_connections  1024;
}
http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;
    server {
        listen       15600 ssl;
        ssl on;
        ssl_certificate     /home/towatt/config/cert/ssl/towatt.com.crt;
        ssl_certificate_key /home/towatt/config/cert/ssl/towatt.com.key;
        server_name  localhost;
        location / {
            auth_basic "secret";
            auth_basic_user_file /opt/nginx/nginx-es/passwd.db;
            proxy_pass http://192.168.5.242:5601;
            proxy_set_header Host $host:5601;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header Via "nginx";
       }
      access_log off;
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }
    }
}

1.2 如果系统没有htpasswd 工具 先安装httpd （注意不要启动httpd）
yum -y install httpd

1.3 生成密码文件 htpasswd -c /opt/nginx/nginx-es/passwd.db username 根据提示输入密码
创建第二个用户 不要加 -c 参数，否则会覆盖掉第一个用户。
重新加载nginx /opt/nginx/nginx-es/nginx -s reload

1.4 重新登陆 访问nginx地址
(https://192.168.5.242:15601) 要求输入 用户名 密码
